2. LUỒNG XỬ LÝ PHIẾU SỬA CHỮA (Workflow Chính)
─────────────────────────────────────────────────────────────────

      │
      ▼

      │
      ▼
[Kỹ thuật viên] Nhận phiếu ─────────► Trạng thái: repairing
      │
      ├── Cần linh kiện? ──YES──► Gửi YeuCauLinhKien
      │                                │
      │                                ▼
      │                    [Thủ kho] Duyệt / Từ chối
      │                                │
      │                    Trạng thái: waiting_parts / repairing
      │
      ▼
[Kỹ thuật viên] Sửa xong ───────────► Trạng thái: done
      │
      ▼
[QA / TraXac] Kiểm tra chất lượng
      │
      ▼
[Support] Trả máy cho KH ───────────► Trạng thái: delivered
      │
      ▼
SMS + Email thông báo cho KH


3. LUỒNG XÁC THỰC (Authentication)
─────────────────────────────────────────────────────────────────

[MVC] Đăng nhập ──► AccountController
         │
         ▼
   POST /api/auth/login
         │
         ▼
   API xác thực NguoiDung + Tenant
         │
         ▼
   Trả về user info + roles
         │
         ▼
   MVC lưu Cookie (30 ngày, sliding)
         │
         ▼
   Mọi request tiếp theo:
   MVC ──► [X-Caller-Email header] ──► API
                                        │
                                        ▼
                              Middleware resolve roles
                                        │
                                        ▼
                              [Authorize] check pass

5. PHÂN QUYỀN NGƯỜI DÙNG
─────────────────────────────────────────────────────────────────

  Role          │ Quyền hạn
  ─────────────────────────────────────────────────────────────
  SystemAdmin   │ Toàn bộ hệ thống, quản lý chuỗi cửa hàng
  StoreAdmin    │ Quản lý 1 cửa hàng, xem báo cáo
